> **用途**：2026-07-15 與老師討論碩論主軸的 high-level 文件。本文只保留討論需要的高度；每個判斷的技術細節與查證證據見文末 §7 的出處對照。
> **整理日期**：2026-07-13
> **一句話**：白板線（otUE + PQC 換槽、重構 UE）的正解已被 HKNY24（TCC 2024）附錄先行做掉，但精讀換來兩項資產——槽位介面的精確刻畫、一套現成的證明模板——由此長出兩條互補的候選主軸：**Unclonable IPFE**（造新原語，細粒度功能）與 **Unclonable IBE**（填空白格，存取結構）。希望這次 meeting 定下主軸與各自的第一個技術決策。

## 0. 一頁總覽

**兩條路線從同一個地方分岔出來。** AK21 配方 `ct = (古典加密(一次性金鑰 k), otUE 加密(訊息 m))` 的古典槽，經精讀刻畫出邊界規則：

```
              AK21 / HKNY24 配方：ct = ( 古典側裝一次性金鑰 k ，量子側 otUE 裝訊息 m )
                                        │
        槽位規則：古典側可以換成任何「all-or-nothing」加密的可模糊化（NC）版本
                                        │
      ┌─────────────────────────────────┴──────────────────────────────────┐
 放進更豐富的 all-or-nothing 加密（IBE）                放進細粒度 FE（IPFE）
 → 配方照走，輸出該存取結構的 unclonable 版          → 功能金鑰只該解出 f(k)，功能性斷裂
 → 【路線二：Unclonable IBE】                        → 必須改造目標原語、重新設計構造
                                                     → 【路線一：Unclonable IPFE】
```

|  | 路線一：Unclonable IPFE | 路線二：Unclonable IBE |
|---|---|---|
| 要做什麼 | 首個「受限函數類」的不可複製 FE | 文獻中不存在的 unclonable IBE |
| 一句話構造 | 向量藏在一次性 pad 下交給古典 IPFE，pad 用 UE 加密 | AK21 配方的古典槽換成 RNC-IBE |
| 為什麼可以 | MM24 掛網 21 個月、零構造跟進；直接合成路徑歸約乾淨 | 槽位規則保證放得進去；證明可套 HKNY24 現成模板 |
| 最硬的點 | 合法解密者會學到超過 ⟨x,y⟩ 的資訊 | 後量子版 RNC-IBE 零件不完整 |
| 保底 | 單金鑰版仍成立「首個」；另有 no-go 短文在手 | 定義 + poly-ID 版定理即可成章 |

## 1. 共同背景（三分鐘版）

1. **AK21（TCC 2021）的配方**：可重用 UE = otUE（BB84 共軛編碼的一次性不可複製加密，量子核心）+ 古典加密（槽）。量子側裝訊息、古典側裝一次性金鑰，KEM-DEM 式拼裝。
2. **古典槽不能是普通加密**：不可複製遊戲的第二階段會把解密金鑰亮給分裂後的兩個對手，語意安全就此作廢。證明需要古典側「可模糊化」——歸約時先把古典密文換成與金鑰無關的假密文，亮金鑰時再交出一把能把假密文「解釋成正確金鑰」的偽金鑰。
3. **精讀的歸納**：證明真正消耗的只是「有 trapdoor 的可模糊化」，恰好就是 RNCE（接收者非承諾加密）的標準介面；AK21 自己寫的偽金鑰定義（Def 16，無 trapdoor）其實過強。
4. **查證結果**：這個觀察是對的——對到 HKNY24（TCC 2024）Appendix E 已經用 RNCE 填槽做出公鑰 UE，連 unclonable-IND 的保持都證了。**白板線的原形（換槽重構 UE）確定關閉。**
5. **轉化下來的資產**：(a) 上圖的槽位規則（吃 all-or-nothing、吃不下細粒度 FE）；(b) HKNY24 App E 的歸約管線——「把古典秘密附在 B/C 兩個 register 上、金鑰揭示後各自本地重建偽金鑰」——是兩條新路線共用的證明模板；(c) 一個已完成的副產品：Clifford 直構的可逆性洩漏攻擊（no-go observation，2–3 頁可交付）。

## 2. 路線一：Unclonable IPFE（repo 代號：路線甲）

### 2.1 要做什麼

造一個新原語：**內積函數加密（IPFE）的不可複製版**。

- **功能**（與古典 IPFE 相同）：訊息是向量 x，函數金鑰對應向量 y，解密只得到內積 ⟨x, y⟩，不洩漏 x 的其他資訊。
- **新性質**：密文含量子態。把一份密文分裂給 B、C 兩人，即使兩人事後各自拿到（獨立生成的）函數金鑰，也無法**同時**解出正確答案——單人可以解（那是功能性），兩人同贏被 no-cloning 擋住。
- **語義定案**：訊息與輸出都是古典的，「量子」只在密文——和 UE 一脈相承（BL20/AK21 的訊息也是古典的）。
- **應用直覺**：資料以量子密文出售，買到函數金鑰的人可以算指定的統計量，但無法把「還能解的密文」複製給第二個人同時使用——不可複製的資料市場 / 隱私統計。

**產出形態**：安全定義（cloning game 特化到內積類）＋ 一條 compiler 定理（「任何 unclonable-IND 安全的 UE × 任何 IND 安全的 IPFE ⇒ unclonable IPFE」）＋ 2–3 個實例化 ＋ 與 MM24 的效率對照（密文從 garbled universal circuit 縮到 O(n log q) 古典 + O(n log q) qubit）。

### 2.2 為什麼可以

**新穎性（已查證）**：
- unclonable FE 目前唯一的論文是 MM24：做的是「所有多項式大小電路」，工具很重（量子 garbled circuits + universal circuit 四層階梯）。它掛網 21 個月仍是預印本，引用僅約五篇、**零構造端跟進**——「受限函數類」這一格完全沒人碰。
- BMMS26（2026-01）證明 SIM 安全的 QFE 不可能 ⇒「受限類 + IND 安全」是被官方劃出來的活路，論文動機不用自己辯護。
- 結構原因：會 lattice FE 的人不熟量子編碼、會量子的人不熟 IPFE 代數——這題落在兩個社群的縫隙，恰好是我們兩邊都讀過的位置。

**可行性（已初勘）**：
- 存在一條**繞過 MM24 整座重型階梯**的直接合成路徑（P1，見下）。精讀同時發現：MM24 的 lifting 定理消耗 universality，對 IPFE「不適用 as stated」——所以換底層不是換零件、是重新設計，這既解釋了為什麼這格空著，也是我們的貢獻空間。
- P1 的歸約 shape 乾淨：對挑戰訊息 x₀, x₁，取 pad 滿足 r₁ = r₀ + x₀ − x₁，古典 IPFE 側在兩個世界完全同分布，整題化約成 UE 的 unclonable-IND。
- UE 歸約段有 HKNY24 App E 的現成模板；元件全部現成（ALS16 的 LWE-IPFE、AKLL22 的 QROM unclonable-IND UE、HKNY24 的明文擴展）。

### 2.3 大概的方式

**構造（P1，直接合成）**：

```
Enc(x):        抽一次性 pad r
               ct = ( IPFE.Enc(x + r) ,  UE.Enc(k, r) )
Dec(sk_y, ct): 先從量子側解出 r，再算 IPFE.Dec(sk_y, ·) − ⟨r, y⟩ = ⟨x, y⟩
```

把 x 用 pad 蓋住丟給古典 IPFE（古典部分隨便複製都不影響安全），不可複製性整個externalize到 r 的 UE 加密上——要算內積就必須先從量子側解出 r，而兩個分裂的對手做不到。

**三步走**：(1) 定義章：把 MM24 的 UFE 安全遊戲特化到內積類；(2) 構造與兩段歸約（IPFE 段＋UE 段，後者套模板）；(3) AK21 式 PRF 可重用化＋實例化表。已有 12–16 週的執行計畫與退場條件（Deep Dive §6）。

**最硬的點（誠實揭露，也是要跟老師討論的核心）**：合法解密者從量子側解出**整個 r** 之後，配上古典側的 x + r 就學到**整個 x**——超過 ⟨x, y⟩。這是「功能性 vs 不可複製性」的真張力，目前兩條出路：

- **保底**：把功能性弱化為單金鑰／有界金鑰版本——作為「首個 unclonable 受限類 FE」的定理仍然成立，論文站得住；
- **進取**：量子側不放裸 r，改放「只能解出 ⟨r, y⟩」的結構（遞迴一層 FE，長得像 MM24 的 trojan 結構）——難度高、價值大。

另有正宗重建階梯的路徑（P2）作為後備，不是首攻。

**競速**：唯一要防的是 MM24 原作者團隊；其 2026-01 的產出是不可能性方向（非構造），判斷每季複查一次。

## 3. 路線二：Unclonable IBE（repo 代號：W1）

### 3.1 要做什麼

造一個文獻中不存在的原語：**不可複製的身分基加密（IBE）**。

- **功能**（與古典 IBE 相同）：寄件人用收件人的身分字串（如 email）直接加密，收件人向金鑰機構領取身分金鑰後解密。
- **新性質**：密文不可複製。拿到「對身分 id* 加密」的密文的對手分裂成 B、C，即使兩人事後都拿到 sk_{id*}、且過程中可以查詢任意多**其他**身分的金鑰，也無法同時解出。
- **定位**：HKNY24 佔掉了 PKE 這格；IBE 是 PKE 之後最基本的存取結構——這是把不可複製性帶進「有金鑰管理結構」的世界的第一步。

**產出形態**：cloning game 定義（身分金鑰查詢怎麼進遊戲）＋ 主定理 ＋ 後量子實例化 ＋ 與 KN23 的分離比較。加分項見 3.3 的一般化定理。

### 3.2 為什麼可以

**新穎性（2026-07-13 查證）**：
- 「unclonable IBE」一詞在文獻中**不存在**（ePrint 全文檢索＋引用追蹤）。
- 最接近的 KN23（TCC 2023）只做「一對多」的弱化安全概念（predicate encryption 設定）；AK21 型完整 cloning game 的 IBE 版沒有被覆蓋。
- 關鍵零件 **RNC-IBE 剛出爐**（Goyal–Kitagawa–Koppula–Nishimaki–Rajasree–Yamakawa，PKC 2025），其論文的應用清單只有 incompressible IBE——**零量子應用**。零件與應用之間這條線還沒人畫。

**可行性**：
- 槽位規則已驗證：IBE 是 all-or-nothing 加密，放得進 AK21 配方的古典槽。
- 證明 = HKNY24 App E 模板 ＋ IBE 標準的金鑰查詢 hybrid——在所有候選中**證明風險最低**。

### 3.3 大概的方式

**構造**：

```
Enc(mpk, id, m):  抽一次性金鑰 k
                  ct = ( RNCIBE.Enc(mpk, id, k) ,  otUE.Enc(k, m) )
```

**定理形狀（候選）**：若存在後量子的 RNC-IBE（selective-ID 即可起步）與 one-time unclonable encryption，則存在 unclonable IBE——語意安全承自 IBE、不可複製性無損承自 otUE，且對任意多的非挑戰身分金鑰查詢安全。

**實質工作三塊（這題不是套模板白撿）**：
1. **後量子 RNC-IBE**：現成的主構造用 bilinear pairing（非後量子）；後量子的 relaxed 版只支援**多項式大小的身分空間**。兩個選項：第一條定理誠實地接受 poly-ID 起步；或自造 LWE 上的 selective RNC-IBE（GPV/ABB 陷門＋可模糊化）——後者本身就是論文的技術核心。**→ 需要老師定案。**
2. **定義**：cloning game 中身分金鑰查詢在「分裂前 vs 分裂後」的給法有幾個自然變體，要選定並論證。
3. **與 KN23 的關係**：證明蘊含／分離，劃清貢獻邊界。

**加分項（把題目做大）——一般化主定理**：

> **otUE ＋ 任何 NC 化的 all-or-nothing 加密 ⇒ 該原語的 unclonable 版。**

一條定理統一整族：PKE 版重現 HKNY24（sanity check）、IBE 當主要實例、CCA 版與 registered 版當 corollary，再加「可模糊化是否必要」的分離問題當理論章——這就是一整本碩論的骨架。

**競速（誠實）**：RNC-IBE 作者群 = NTT ＋ Goyal 產線，他們自己補量子應用的能力最強，此題窗口可能比路線一短。若選此線，合理策略是儘快把「定義＋poly-ID 版定理」寫成 note 掛 ePrint 佔位，再做深。

## 4. 兩線對照與論文骨架選項

| 維度 | 路線一 IPFE | 路線二 IBE |
|---|---|---|
| 產出物 | 新原語：細粒度功能 × 不可複製 | 新原語：存取結構 × 不可複製 |
| 新穎性窗口 | 大（21 個月零競爭） | 全空，但窗口可能較短（NTT 產線） |
| 技術風險 | 中-高：解密者洩漏 x 是真研究問題 | 低-中：證明套模板；風險集中在後量子零件 |
| 證明工作量 | 兩段歸約自己組裝 | 模板＋標準 hybrid |
| 保底 | 單金鑰版＋Clifford no-go 短文 | 定義＋poly-ID 版定理 |
| 精讀投資的變現 | MM24 / ABDP15 / AK21 / AKL23 全用上 | AK21 / HKNY24 精讀直接變現 |

**骨架選項（三選一）**：

- **方案 A**：路線一為主軸（12–16 週計畫現成），路線二只做「定義＋第一定理」當論文的第二章或備援。
- **方案 B**：路線二為主軸，以一般化主定理為骨架（PKE／IBE／CCA／registered 一條線收編＋理論章）；路線一收縮為延伸章或不做。
- **方案 C**：先跑 4 週共同前置（兩線的定義語言與證明模板本來就共用，見 §6），拿「路線一 D2 試探結果」與「RNC-IBE 精讀結果」回來再定案。

**我方傾向**：方案 A（理由：競爭窗口最大、保底最厚、計畫現成），但方案 B 的骨架統一性與低證明風險是真實的優點——請老師裁決。
